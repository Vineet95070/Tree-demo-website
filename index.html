Tree

A tree data structure is a hierarchical structure that consists of nodes connected by edges. It is a widely used abstract data type that simulates a hierarchical tree structure, with a root value and subtrees of children, represented as a set of linked nodes. Here are some key features and definitions related to trees:


Binary Tree:

A binary tree is a type of tree data structure in which each node has at most two children, referred to as the left child and the right child.

Key Properties of BST
  1. Left Subtree: The value of each node in the left subtree is less than the value of the parent node.
  2. Right Subtree: The value of each node in the right subtree is greater than the value of the parent node.
  3. No Duplicates: Generally, BSTs do not allow duplicate values, but they can be modified to allow duplicates either on the left or right subtree.
  4. Recursion: BSTs use recursion in their traversal, insertion, and deletion operations.
Operations on a BST
  1. Search:
    • To search for an element, start at the root and recursively traverse the tree.
    • If the element is smaller than the current nodeâ€™s value, move to the left subtree; if larger, move to the right subtree.
    • Time Complexity: O(log n) on average (O(n) in the worst case).

  2. Insertion:
    • To insert a new element, follow the same logic as in the search operation. Find the correct position where the new element should go and insert it as a leaf node.
    • Time Complexity: O(log n) on average (O(n) in the worst case).
  3. Deletion:
    • Deletion can be handled in three cases:
      1. Leaf Node: Simply remove the node.
      2. Node with One Child: Replace the node with its single child.
      3. Node with Two Children: Find the nodeâ€™s in-order successor (the smallest node in the right subtree) or in-order predecessor (the largest node in the left subtree), replace the node with that value, and delete the successor/predecessor.
    • Time Complexity: O(log n) on average (O(n) in the worst case).


Traversal Techniques

  • In-Order Traversal:
    • Traverse the left subtree, visit the node, and then traverse the right subtree. This results in the values being accessed in ascending order.

  • Pre-Order Traversal:
    • Visit the node first, then traverse the left subtree and right subtree.
    • Order: Root â†’ Left â†’ Right
  • Post-Order Traversal:
    • Traverse the left subtree, right subtree, and then visit the node.
    • Order: Left â†’ Right â†’ Root
  • Level-Order Traversal (Breadth-First):
    • Traverse the tree level by level from the root to the leaves. This is typically implemented using a queue.
Advantages of BST
  • Efficient Search Operations: On average, searching, insertion, and deletion in a balanced BST take O(log n) time.
  • Sorted Data: In-order traversal of a BST gives sorted data in ascending order.
  • Dynamic Size: BSTs grow and shrink dynamically, so you donâ€™t need to predefine their size.
Disadvantages of BST
  • Unbalanced Trees: In the worst case, where the tree is skewed (like a linked list), the time complexity for search, insertion, and deletion becomes O(n).
  • Balancing Required: To ensure efficiency, a BST may need to be balanced, which leads to the introduction of self-balancing BSTs like AVL Trees or Red-Black Trees.
Balanced BSTs
  • AVL Trees: A self-balancing BST where the difference in heights between the left and right subtrees of any node is at most 1. Rotations are used to maintain balance after insertions and deletions.
  • Red-Black Trees: A type of self-balancing BST where nodes are colored either red or black, and the tree satisfies specific balancing properties, ensuring that the longest path from the root to a leaf is no more than twice the length of the shortest path.
Applications of BST
  • Database Indexing: BSTs are used in databases for fast retrieval of records.
  • In-Memory Search: Useful for applications that require dynamic ordered data and fast search capabilities.
  • Sorting: BSTs can be used to perform an in-place sort via in-order traversal.
